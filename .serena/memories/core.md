# Core — flux-talos

Home Kubernetes GitOps mono-repo. FluxCD reconciles `kubernetes/` against the cluster; Talos Linux runs on all nodes.

## Top-level layout

```
kubernetes/
  apps/         # all workloads, one dir per namespace (~30 namespaces)
  bootstrap/    # one-time cluster bootstrap manifests
  components/   # reusable Kustomize components (volsync, externalsecret-refresh, psa-privileged, …)
  flux/         # FluxCD cluster entrypoint (cluster/ks.yaml → apps/)
talos/
  talconfig.yaml   # talhelper source-of-truth for all node configs
  clusterconfig/   # generated machine configs (gitignored, regenerated by just talos gen-config)
  talsecret.sops.yaml / talenv.sops.yaml
.renovate/      # modular Renovate config snippets
.github/        # CI/CD workflows
scripts/        # helper shell scripts
```

## Secrets

- SOPS + AGE key at `age.key` (loaded via `SOPS_AGE_KEY_FILE`)
- `kubernetes/.+\.sops\.yaml` → `encrypted_regex: ^(data|stringData)$` + `mac_only_encrypted: true`
- `talos/.*\.sops\.yaml` → full encryption
- External Secrets Operator pulls runtime secrets from Bitwarden Secrets Manager (`bitwarden-sdk-server`)
- `.bws_access_token` file provides local `BWS_ACCESS_TOKEN`

## Storage layers

- Longhorn — replicated PVCs
- OpenEBS — local hostpath
- democratic-csi — NFS/iSCSI from TrueNAS
- Object storage: self-hosted Garage (S3-compatible on TrueNAS) for Thanos, CNPG backups, etcd backups, Volsync local
- Volsync daily offsite to Cloudflare R2
